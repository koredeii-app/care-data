# care-data プロジェクト指針

## データ構造
- `data/` 以下に都道府県別JSONファイル
- `data/citylinks.json`: 市区町村名 → 公式センター一覧ページURL
- JSONフィールド: `{id, prefecture, city, area, name, tel, url}`（IP電話がある場合は `tel_ip` を追加）
- ID は全ファイル横断でユニーク

## URLルール
- **公式ページに各センターの個別リンクが掲載されている場合は必ず個別URLを使用する**
- 個別リンクが存在しない場合のみ一覧ページURLを使用する
- 独自に調べたURLは使わない。公式ページに載っているURLのみ使用する

## データ収集ルール
- `area` フィールド: 公式ページに担当町名が掲載されている場合は町名一覧を使用する
- 電話番号に市外局番が省略されている場合は補完して記載する
- センター名は公式表記に従う

## 動作方針
- トークン使用を最小限に抑える
- ファイルは必要な行範囲だけ読む（offset/limit 活用）
- Grep/Glob を使えるなら Read より優先
- 編集後の確認読み込みは不要
- 節約できる機会があれば都度提案する
